# PlayUp — Спецификация лендинга 2026

**Для дизайнеров и разработчиков. Полный документ: эстетика, копирайт, визуальная иерархия, дизайн и движение по каждому из 9 блоков.**
Версия запуска · Белград

---

## Как читать документ

- Сначала — **эстетическое направление** и **дизайн-система**. Это фундамент, на котором держится «2026, а не 2016». Прочитать до блоков.
- Дальше — **9 содержательных блоков** в порядке прокрутки. Каждый по схеме: *назначение → композиция → копирайт (финальный) → иерархия → дизайн → движение*.
- Между блоками и поверх них — **слой призыва к действию** (CTA) и **конструктив** (шапка, форма записи, подвал). Это не отдельные блоки повествования, а сквозные элементы — описаны в конце.
- В конце — **технический стек**, **фото**, **производительность**, **доступность**, **чек-лист сдачи**.

Весь копирайт — финальный, берётся как есть. Английский — только в фирменной подписи «Belonging through play.» и в технических терминах.

---

## Эстетическое направление (главное)

**Кинематографичный редакционный журнал в движении.** Тёплый, плотный, уверенный. Не «чистый стартап-лендинг», не презентация из блоков. Скорее — лонгрид глянцевого журнала о жизни (Aesop, Kinfolk, Soho House), который листается как кино.

Три кита, на которых это держится:

1. **Фото — это среда, а не картинка.** Кадры живут от края до края, текст идёт поверх них, человек оказывается внутри сцены. Никаких фото в маленькой рамочке с большими полями.
2. **Движение — это атмосфера, а не украшение.** Плавная инерционная прокрутка, фото закрепляется и текст проходит поверх, слои движутся с разной скоростью, заголовки проявляются по строкам. Образ жизни продаётся движением.
3. **Тёплая темнота и крупная типографика.** Глубокие зелёные и угольные секции с плёночным зерном, чередующиеся со светлыми. Огромный Cormorant как графика. Левое выравнивание, асимметрия, намеренная плотность.

**Что мы сознательно НЕ делаем** (маркеры устаревшего): карточки с тенями, единственная анимация «выплыло снизу», центрованная симметрия, статичный герой, сток-иконки и чёрные фигуры-заглушки, фейковые таймеры обратного отсчёта. Подробности — в дизайн-системе.

## Дизайн-система

### Цвета (из брендборда)

```css
--bone:        #F7F6F2;   /* основной светлый фон */
--beige:       #EDE5DA;   /* тёплые светлые секции */
--stone:       #D8D2C7;   /* разделители, тонкие линии */
--green:       #2F4A3F;   /* тёмная секция, кнопки, шапка */
--green-deep:  #233A30;   /* самый глубокий зелёный для контраста слоёв */
--charcoal:    #1C1C1C;   /* самые тёмные секции */
--clay:        #C97A5D;   /* единственный акцент — точечно */
--text:        #3A3A3A;
--muted:       #8A857C;
--on-dark:     rgba(255,255,255,0.92);
--on-dark-dim: rgba(255,255,255,0.55);
```

Правило акцента: **clay редко и точечно** — подчёркивание у заголовка, надстрочник, одна иконка, активный элемент. Больше трёх clay-пятен на экране — перебор.

### Типографика

```css
/* Дисплейный — заголовки, крупные акценты, цифры-графика */
font-family: 'Cormorant Garamond', serif;   /* 400 / 500 / 600 */

/* Текстовый — весь остальной текст, надстрочники, кнопки */
font-family: 'DM Sans', sans-serif;          /* 400 / 500 / 700 */
```

Масштаб (десктоп → мобайл):
```
H1 (hero):        clamp(64px, 9vw, 132px)   — Cormorant, line-height 0.98
H2 (секции):      clamp(40px, 5.5vw, 84px)  — Cormorant
H3 (карточки):    clamp(22px, 2.4vw, 34px)  — Cormorant
Цифра-графика:    clamp(80px, 16vw, 240px)  — Cormorant 500
Надстрочник:      12–13px DM Sans, КАПС, letter-spacing 0.14em, --clay
Тело:             clamp(16px, 1.15vw, 19px) DM Sans, line-height 1.65
Подпись:          13–14px DM Sans, --muted
```

Принцип: заголовки крупнее, чем «безопасно». Крупная типографика на цвете — главный носитель премиума. Заголовки секций — **по левому краю**, не по центру (левое читается дороже).

### Сетка и ритм

- Контейнер max-width 1280px; ключевые фото — full-bleed (на всю ширину вьюпорта).
- Поля: 20px (мобайл) / 48px (планшет) / 80px (десктоп).
- **Асимметрия по умолчанию:** блоки смещены, плитки разного размера, крупное рядом с мелким, текст ломает сетку. Идеально ровные сетки 2×2/3×2 — избегать.
- **Плотность и поток:** тёмное–светлое чередуется часто, фон одной секции протекает в следующую (без жёстких «полок»). Не ставить три светлых секции подряд. Светлые блоки компактнее по высоте.

### Скругления и линии

- Скругления — минимальные: 4px у кнопок и полей, 8px у плиток. Крупные «мыльные» радиусы не используем — они дешевят.
- Разделение — тонкой линией `--stone` (1px) или сменой фона, **не тенью**. Drop-shadow не используем нигде.

### Текстуры

- На тёмных секциях (green, charcoal) — лёгкое **плёночное зерно** (SVG/PNG noise overlay, opacity 3–5%, `mix-blend-mode: overlay`). Даёт тёплое аналоговое ощущение вместо плоской заливки.
- На светлых — едва заметная бумажная фактура (опционально, очень тонко).

### Иконки

- **Один набор**, тонкая контурная линия, цвет `--clay`, единый вес штриха. Из фирменного набора: сообщество, ракетка, кофе, календарь, пин, бег, сердце.
- **Запрещено:** сток-иконки в разнобой, эмодзи, чёрные фигуры-заглушки. Если для блока нет осмысленной иконки — лучше без неё.

### Движение (motion-система)

```
Скролл:           Lenis — плавная инерционная прокрутка (базовый слой, на всём сайте)
Закрепление:      GSAP ScrollTrigger — pin в ключевых секциях (фото стоит, текст проходит)
Параллакс:        слои движутся с разной скоростью (фото медленнее, текст быстрее)
Появление текста: крупные заголовки проявляются по строкам/словам (SplitText-подход), с задержкой
Фото:             постоянный лёгкий зум (scale 1.0→1.06 за ~20с) в герое и pinned-кадрах
Hover:            зум фото 1.03; подчёркивание ссылок прорисовывается слева направо
Клик:             scale 0.98 на 100мс
Курсор (десктоп): опционально — точка/кружок, реагирует на интерактив
```

Принцип: движение помогает читать и создаёт атмосферу. Никаких мигающих, прыгающих, перетягивающих внимание дольше секунды элементов. **Единственная анимация «opacity 0→1 + translateY» как универсальный приём — не использовать.** Появление всегда привязано к скроллу и слоям, а не «всё выплывает снизу одинаково».

При `prefers-reduced-motion: reduce` — отключить параллакс, pin, зум, видео (показать постер); оставить мгновенный показ.

## Блок 1 — Hero

**Назначение.** За 5 секунд — что это и про кого. Не объяснять, а попасть в чувство и сразу дать действие.

**Композиция.** Full-bleed живой фон на весь экран (100vh). Текст — **в нижней трети слева**, не по центру, не в отдельной колонке. Человек оказывается внутри сцены, текст лежит поверх. Лёгкое затемнение снизу (градиент от `rgba(0,0,0,0.45)` к прозрачному вверху) — чтобы текст читался.

**Копирайт.**
```
[Надстрочник]
БЕЛГРАД · СОЦИАЛЬНЫЙ СПОРТ

[H1]
Свои через игру.

[Подзаголовок]
Раз в неделю — игра, своя команда и кофейня сразу после.
Приходишь один, уходишь со своими.

[Кнопка]   Записаться на игру →
[Ссылка]   Как это работает ↓

[Живая подсказка, мелко, у кнопки]
Ближайшая игра — в четверг
```

**Иерархия.** H1 доминирует (это самый крупный текст на сайте). Под ним — подзаголовок вдвое мельче. Надстрочник — тонкий маркер сверху. Кнопка — единственный сплошной акцент. «Ближайшая игра — в четверг» — тихая деталь, связывает с записью.

**Дизайн.**
- Фон: видео-петля (люди после игры / кофейня / движение, тёплый вечерний свет) — `muted autoplay loop playsinline`, постер-кадр. Если видео нет — фото с постоянным медленным зумом.
- H1 — Cormorant 500, белый, с тонким подчёркиванием `--clay` под одним словом («Свои»).
- Кнопка — фон `--green` поверх фото (или белая, если фон тёмный), текст белый, скругление 4px.
- **Никаких плавающих карточек поверх героя** (карточку «кофе после игры» — убрать). Чистый кадр.
- Бейдж в углу (опционально, тонко): «Старт 2026» — рамка `--clay`, прозрачный фон.

**Движение.**
- Загрузка: H1 проявляется **по словам** (слова поднимаются по очереди, задержка 60мс), затем подзаголовок, затем кнопка.
- Фон: видео играет / фото медленно зумит.
- При скролле фон уходит с лёгким параллаксом (медленнее контента) — ощущение глубины.

**Мобайл.** Фон 90vh, текст внизу слева. Кнопка — на всю ширину. Видео заменить на постер при медленной сети.

## Связка — бегущая строка (не блок)

Тонкая лента сразу под героем, протекает в следующую секцию. Это связующий элемент, не блок повествования.

```
[Очень медленная горизонтальная бегущая строка, Cormorant, на --green с зерном]
Раз в неделю · Та же команда · Кофейня, не бар · Первая игра бесплатно · Приходишь один — уходишь со своими ·
```
DM-sans-капсом надстрочник не нужен; сами фразы — Cormorant среднего кегля, `--on-dark`. Движение медленное, бесконечное, ненавязчивое. При reduced-motion — статичная строка.

## Блок 2 — Кажется, это про тебя

**Назначение.** Самоидентификация. Боль подаётся через прямую речь — человек узнаёт свою ситуацию и говорит «о, это я». Самый конвертящий блок узнавания.

**Композиция.** Split-pin на десктопе: **слева закреплён заголовок** (стоит на месте), **справа прокручиваются плитки-цитаты** одна за другой. На мобайле — вертикальная лента плиток.

**Копирайт.**
```
[Надстрочник]   ДЛЯ КОГО
[H2]            Кажется, это про тебя.

[Плитки — каждая: крупная цитата + строка + тег]

«Переехал. Пока никого.»
Знакомишься только на работе, выходные — кофе в одиночку.
Не нетворкинг нужен, а просто свои.
— Новенький в городе

«Плачу за зал. Не хожу.»
Дорожка, наушники, домой. Спорт вроде есть — а людей рядом нет.
— Устал от зала

«Друзья разъехались.»
Хотел бы команду. Но писать в десять чатов и всех собирать — увольте.
— Ищу своих

«Вместе — а заняться нечем.»
Ей хочется движения, он согласен, только если не скучно.
Тут не скучно обоим.
— Вдвоём

«Последний раз играл в универе.»
Давно ничего не делал с телом. Страшно начинать?
Тут все такие же. Правда.
— Начинаю заново

«В Белграде меньше года.»
Хочется людей, а не визиток и дежурной болтовни.
— Релокант
```

**Иерархия.** В каждой плитке главное — **цитата** (Cormorant H3, как заголовок). Под ней пояснение (DM Sans, `--text`). Внизу тег-чип (мелкий, `--green` на `--green-light`). Заголовок секции слева — крупный якорь, держит весь блок.

**Дизайн.**
- Фон секции `--bone`.
- Плитки — **плоские**, на `--beige`, скругление 8px, **без тени**, без иконок. Воздух внутри.
- Асимметрия: плитки чуть разного размера/смещения, не идеальная сетка.
- Hover: тонкая рамка `--clay` прорисовывается по контуру.

**Движение.** Заголовок слева закреплён (pin), пока справа плитки въезжают по очереди (параллакс/проявление, не «выплыванием снизу»). На мобайле — плитки проявляются при входе в экран с малой задержкой.

**Мобайл.** Заголовок сверху (не закреплён), плитки вертикально на всю ширину.

## Блок 3 — Всё уже устроено

**Назначение.** Снять страх «я приду, а там надо что-то решать, организовывать, и будет неловко». Это про **заботу и отсутствие трения** — тебя встречают, всё готово. (Отдельно от уровня — тот про матчмейкинг, блок 7.)

**Композиция.** Тёмная секция (`--green-deep` с зерном). Крупное типографическое заявление слева, одно поддерживающее фото справа (или фоном) — хост встречает игрока, тёплый момент.

**Копирайт.**
```
[Надстрочник]   НИЧЕГО НЕ НАДО РЕШАТЬ
[H2]            Просто приходи.
                Остальное — на нас.

[Тело]
Не нужно искать соперников, бронировать корт, собирать людей
и держать живым ещё один чат. Ты оставляешь заявку — а дальше
тебя встречает хост: знакомит с командой, ведёт игру, следит,
чтобы никто не стоял в стороне.

От тебя — только прийти.
```

**Иерархия.** H2 («Просто приходи. Остальное — на нас.») — крупный, на две строки, доминирует. Тело — спокойное, узкая колонка (max-width 460px). Финальная строка «От тебя — только прийти.» — отдельно, чуть крупнее тела, как точка.

**Дизайн.**
- Фон `--green-deep`, зерно, текст `--on-dark`.
- Фото хоста — edge-to-edge справа (половина экрана) или мягко в фоне за текстом.
- Иконка-сообщество `--clay` рядом с надстрочником (опционально).

**Движение.** H2 проявляется по строкам. Фото справа — лёгкий параллакс. Финальная строка появляется с задержкой после остального текста.

**Мобайл.** Фото сверху (edge-to-edge), текст под ним.

## Блок 4 — Это не одна игра. Это твоя неделя

**Назначение.** Показать, что PlayUp — комьюнити, а не разовый матч. Тезис + доказательство: вокруг игры собрана вся неделя. Здесь же закладывается ценность будущей подписки (одной спокойной строкой).

**Композиция.** Светлая секция. Крупное заявление слева вверху, под ним — **горизонтальная лента** активностей недели (или асимметричная раскладка плиток), которая прокручивается вбок при вертикальном скролле (pinned horizontal) на десктопе.

**Копирайт.**
```
[Надстрочник]   НЕ РАЗОВАЯ ИГРА
[H2]            Это не одна игра.
                Это твоя неделя.

[Тело]
PlayUp не заканчивается финальным свистком. Вокруг игры —
то, из чего собирается неделя. Мы продумали это за тебя:
тебе остаётся выбрать, куда пойти.

[Плитки активностей — иконка + заголовок + строка]

Воскресный бег
Пять километров в темпе разговора и кофе на берегу.

Субботние тусовки
Раз в неделю — без мяча и ракеток. Ужин, кино, вылазка за город.

Ивенты и встречи
Турниры, гости, поводы собраться чаще, чем раз в неделю.

Восстановление
Сауна, массаж, растяжка. То, до чего сам обычно не доходишь.

[Спокойная строка под лентой]
Со временем всё это соберётся в подписку — запустим, когда
наберётся костяк.
```

**Иерархия.** H2 — крупный якорь. Плитки активностей равны между собой, но второстепенны относительно заголовка. Строка про подписку — мелкая, `--muted`, тихая (это единственное упоминание подписки на лендинге, кроме FAQ).

**Дизайн.**
- Фон `--bone` или `--beige`.
- Плитки — плоские, без тени, тонкая иконка `--clay` сверху, единый набор. Между плитками — воздух или тонкая линия `--stone`.
- Если горизонтальная лента — виден край следующей плитки (намёк, что листается).

**Движение.** На десктопе — секция закрепляется, лента активностей едет горизонтально по мере вертикального скролла (pinned horizontal scroll, GSAP). На мобайле — обычный горизонтальный свайп со «снепом». H2 — по строкам. При reduced-motion — статичная сетка.

**Мобайл.** Заголовок, под ним лента активностей горизонтальным свайпом, строка про подписку под ней.

## Блок 5 — Лучшее после свистка (кафе)

**Назначение.** Эмоциональное ядро и главный отстройщик. Продукт — не спорт, а люди и стол после игры. Самый кинематографичный момент страницы.

**Композиция.** **Pinned full-bleed фото** кофейни: кадр закрепляется на весь экран, текст проходит поверх него по мере скролла. Тёмно, тепло, вечер.

**Копирайт.**
```
[Надстрочник]   ПОСЛЕ ИГРЫ
[H2]            Лучшее — после
                финального свистка.

[Тело]
Каждый раз всё заканчивается одинаково: после игры все идут
в одну кофейню. Сорок минут, нормальный кофе, разговоры ни о чём.

Игра — это повод. Этот стол — причина, по которой
возвращаешься через неделю.

[Плашка партнёра]
КОФЕЙНЯ-ПАРТНЁР
Good Thanks Café · Дорчол · −20% участникам
```

**Иерархия.** H2 поверх фото — крупный, белый, доминирует. Тело — узкая колонка поверх затемнённой части кадра. Плашка партнёра — мелкая, тонкая, внизу.

**Дизайн.**
- Фото edge-to-edge, закреплено. Затемнение под текстом (`rgba(0,0,0,0.4)`), чтобы читалось.
- На стакане в кадре — логотип PU (если есть такой кадр).

**Движение.** Фото закрепляется (pin), текст проявляется по строкам и проходит поверх. Лёгкий зум фото всё время закрепления. Это пиковый «вау-момент» — на нём не экономить.

**Мобайл.** Фото на 80vh, текст под ним или поверх (если читается). Pin можно упростить до параллакса.

# Конструктив и слой действия

Это не блоки повествования, а сквозные элементы. Они обеспечивают, что запись на игру — главное действие, доступное всегда.

## Шапка (sticky)

```
Слева:   логотип PlayUp (белый знак)
Центр:   Как устроено · Сообщество · Цена · Вопросы   (anchor-ссылки)
Справа:  [ Записаться на игру → ]
```
- Фон `--green`, высота 64px (десктоп) / 56px (мобайл), прилипает к верху, лёгкая тень при скролле >100px.
- Кнопка — единая формулировка на всём сайте.
- Мобайл — гамбургер, полноэкранное меню на `--green`, та же кнопка крупно.

## Слой призыва к действию (CTA)

Двухслойный — как договорено:

1. **Липкий CTA — всегда виден.** На десктопе — кнопка в шапке. На мобайле — узкая нижняя полоса (`--green`) с кнопкой «Записаться на игру →», появляется после первого экрана, ведёт к форме. Ловит тех, кто решил в любой момент.

2. **Контекстные дожимы — через каждые ~2 блока, с разной формулировкой под контекст.** Не одинаковая кнопка в лоб (это раздражает и дешевит), а лёгкий элемент — иногда кнопка, иногда текст-ссылка с прорисовывающимся подчёркиванием. Примеры размещения и формулировок:
   - После блока 4 (комьюнити): «Хочешь так же? Запишись на ближайшую игру →»
   - После блока 5 (кафе): «Прийти на игру — и остаться на кофе →»
   - После блока 7 (анти-обещание): «Если это про тебя — тебе сюда →»
   - Все ведут в форму записи.

**Чего не делаем:** фейковых таймеров «осталось 2 часа», агрессивных всплывающих окон. Реальный остаток мест (если показываем) — честный стимул, этого достаточно.

## Финальная секция записи (точка конверсии)

Это конверсионный финал — куда ведут все кнопки. Здесь живёт «старт скоро / запишись одним из первых» и выбор спорта.

**Композиция.** Тёмная секция (`--green` с зерном), по центру или со смещением, лёгкий угловой паттерн.

**Копирайт.**
```
[Надстрочник]   СТАРТ СКОРО · БЕЛГРАД
[H2]            Запишись одним из первых.

[Тело]
Выбери спорт — мы напишем, когда соберётся твоя ближайшая игра.

[Выбор спорта — чипы]
Падел     Футбол     Волейбол — скоро

[Форма]
[ Твоя почта ]   [ Записаться на игру → ]

[Микрокопия]
Без спама. Только дата ближайшей игры и подтверждение места.

[Живая строка, если есть данные]
Ближайший падел — четверг, 19:00. Осталось 3 места.
```

**Иерархия.** H2 крупный, белый. Чипы спорта — заметные, кликабельные (выбор подбирает ближайшую игру и повышает релевантность). Поле + кнопка — крупные. «Волейбол — скоро» неактивен, помечен честно. Живая строка с остатком мест — тихая, но конкретная (двигатель «записаться сейчас»).

**Дизайн.**
- Поле — рамка `rgba(255,255,255,0.3)`, фон полупрозрачный, фокус → рамка `--clay`.
- Кнопка — белая, текст `--green`.
- Чип «Волейбол — скоро» — приглушён, рамка `--clay`.
- Остаток мест — реальный, из системы записи. Нет данных — строку скрыть.

**Поведение.** Валидация почты на клиенте. После отправки — поле/кнопка заменяются на «Готово. Напишем про ближайшую игру по твоему спорту.» без перезагрузки.

## Технический стек

Текущий стек (Next + Tailwind + Framer Motion) недостаточен для motion-first — на нём и получается «fade-in 2016». Обязательно добавить:

```
Скролл:           Lenis — плавная инерционная прокрутка (база, на всём сайте)
Скролл-эффекты:   GSAP + ScrollTrigger — pin (блоки 5, и горизонталь в 4/6), параллакс, scrub
Появление текста: SplitText-подход (GSAP) — заголовки по словам/строкам
Микро-интеракции: Motion (Framer Motion) — hover, клики
Видео-герой:      self-hosted mp4 + webm, постер, ленивая подгрузка
Текстура:         noise-overlay (SVG/PNG) на тёмных секциях, opacity 3–5%
Форма:            Brevo / Mailchimp / Resend (сбор почты + выбор спорта)
Доступность:      полный fallback при prefers-reduced-motion
Деплой:           Vercel · Аналитика: Vercel Analytics + Hotjar (тепловые карты)
```

**Критично для разработчика:** ядро эффекта — Lenis (плавный скролл), ScrollTrigger (pin + параллакс + горизонтальный скролл), живой герой и появление заголовков по строкам. Если свести это обратно к простому «opacity при появлении» — сайт снова станет «2016», и весь смысл спецификации теряется. Эти вещи — обязательны, не опциональны.

## Фотографии (сквозное)

- **Только живые реальные люди**, естественный свет, размытый городской фон, настоящие эмоции. Никакого глянцевого стока с натянутыми улыбками и идеальными атлетами — это противоречит позиционированию «обычные, давно не игравшие люди» и считывается как искусственное.
- Для запуска допустима временная съёмка; **после пилота заменить на реальные кадры с первых игр**, даже неидеальные. Для этого бренда живое несовершенное фото конвертит лучше, чем красивое искусственное.

Нужные кадры:
```
□ hero (видео-петля или фото)  — люди после игры / движение / вечерний свет
□ host                          — хост встречает игрока (блок 3)
□ week ×4                       — бег, тусовка, ивент, восстановление (блок 4)
□ cafe                          — стол, стаканы (лого PU), руки, разговор (блок 5)
□ closing                       — атмосфера, опционально фоном (финал)
```

## Производительность

Трафик в основном мобильный (Instagram), скорость критична:
- LCP < 2.5 сек. Видео-герой: постер первым, видео догружается.
- Видео ≤ ~3 МБ (webm + mp4). Фото — WebP, ленивая загрузка ниже первого экрана.
- GSAP/ScrollTrigger и Lenis — только где нужно, не вешать тяжёлые эффекты на весь документ без надобности.
- Тестировать на среднем Android, не только на флагмане.

## Доступность

- Контраст AA (особенно белый на зелёном и clay). Видимый фокус у всех интерактивных элементов, навигация с клавиатуры.
- Осмысленный alt у фото на русском. Форма: label связан с полем, ошибки через aria-live.
- При `prefers-reduced-motion: reduce` — отключить параллакс, pin, зум, горизонтальный скролл и видео (постер); оставить мгновенный показ. Контент полностью доступен без движения.
